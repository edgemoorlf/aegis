# Aegis

> A data-centric, capability-based substrate for multi-agent systems.
> **Protect the data, not the sandbox.**

[![CI](https://github.com/edgemoorlf/aegis/actions/workflows/ci.yml/badge.svg)](https://github.com/edgemoorlf/aegis/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## Why

The prevailing approach to agent safety is **execution sandboxing** — isolate the
process or VM so misbehaving code can't harm the host. That's the wrong primary
defense:

- **It protects the wrong layer.** The asset worth protecting is *data*, not the host OS.
- **The breach is often upstream.** If context is sent to an external LLM to generate code, the sensitive data already left the trust boundary before any sandbox runs.
- **It ignores multi-agent consistency.** Per-agent isolation makes coherent shared state *harder*, not easier.

Aegis inverts the model: **the data layer is the security and consistency boundary.**
Compute is assumed untrusted and transient. Data is immutable, versioned, and
access is governed by capabilities — which can now be *generated by an LLM and
verified by a verifier agent*, removing the historical reason capability-based
security never scaled.

See [PLAN.md](PLAN.md) for the full architecture and roadmap.
See [THREAT_MODEL.md](THREAT_MODEL.md) for the threat analysis (exfiltration, prompt injection, cross-agent divergence, capability escalation).

## Quickstart

```python
from aegis import ImmutableStore, AuditLog

store = ImmutableStore()        # append-only, versioned
audit = AuditLog()              # tamper-evident ground truth

v1 = store.append("ledger", "balance", {"amount": 100}, author="planner")
v2 = store.append("ledger", "balance", {"amount": 80}, author="executor")
audit.append("executor", "data.append", "ledger/balance", details={"version": v2.version})

# Nothing is ever lost — time-travel to any prior version
assert store.read("ledger", "balance", as_of_version=1).value == {"amount": 100}

# "Deletion" is a logical tombstone; history is preserved
store.tombstone("ledger", "balance", author="rogue")
assert len(store.history("ledger", "balance")) == 3

# The audit chain is tamper-evident
assert audit.verify() is True
```

Phase 2 adds `SQLiteCapabilityBroker` and `EnforcedStore` — deny-by-default
enforcement with a full audit trail:

```python
from aegis import (
    ImmutableStore, AuditLog, Capability, DataSelector,
    SQLiteCapabilityBroker, EnforcedStore,
    READ, APPEND, TOMBSTONE, AccessDeniedError,
)
import time

store  = ImmutableStore()
audit  = AuditLog()
broker = SQLiteCapabilityBroker(audit, secret=b"keep-this-secret")
estore = EnforcedStore(store, broker, audit)

# Issue a scoped capability token to an agent
cap = Capability(
    selector=DataSelector("ledger"),
    operations=frozenset({READ, APPEND}),
    agent_id="agent-a",
    issued_by="admin",
    not_after=time.time() + 3600,
)
token = broker.issue(cap)

# All access goes through the enforced store
v = estore.append(token, "ledger", "balance", {"amount": 100})
assert estore.read(token, "ledger", "balance").value == {"amount": 100}

# Revoke immediately; subsequent access is denied
broker.revoke(cap.capability_id, by="admin")
try:
    estore.read(token, "ledger", "balance")
except AccessDeniedError:
    pass  # denied — as expected

# Attenuation: derive a read-only token without contacting the issuer
read_token = broker.issue(cap.attenuate(operations=frozenset({READ})))
assert estore.read(read_token, "ledger", "balance").value == {"amount": 100}

# The audit chain covers every issuance, check, and data operation
assert audit.verify() is True
```

## Design Principles

1. **Never delete.** Append-only, versioned, time-travel. Deletion is a logical tombstone.
2. **Data-centric, not action-centric.** Protect data at the storage/access layer, independent of what code runs.
3. **Capability-based access.** No valid token, no access — regardless of executed code.
4. **LLM-generated, agent-verified capabilities.** Removes the manual-configuration burden that sank classical capability security.
5. **Audit log as ground truth.** Agent memories are derived views; the log is authoritative.
6. **Local-first inference.** The LLM lives inside the trust boundary by default.
7. **Versioned memory with provenance.** Divergence between agents is detectable.
